  • question_answerHow can you avoid a return of same coins when no data is saved?
    As mentioned, we do not save any data in our database, such as deposit and withdrawal addresses or transactions. Such information is only used as long as is technically necessary to complete a transaction and is automatically deleted after. To ensure that our users do not receive the same coins they have mixed earlier, Smartmixer uses a One-Way-Encryption process with hashing operations.

    Different parameters will be utilized in order to calculate the hashes. The result of the one-way-encryption algorithm will be assigned to the smart code and cannot be deduced from our deposit address. Therefore, the smart code should be entered before each mixing operation.

    Additionally, we assign our users one of several pools for each transaction. So, whenever the user has been already assigned a specific Pool for a transaction, he or she will receive withdrawals only from other pools of the same category that are managed internally in our system. Thus, we guarantee the highest security in just two steps and without loss of anonymity. Our users never receive the same coins mixed before with our service.
